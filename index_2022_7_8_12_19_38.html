Notion 페이지를 Web Front로 사용하는 자동화 tool

Notion Page를 Web Front로 사용하는 자동화 tool 만들기

1. 기획

1-1. 아이디어 카피

  • 우피라는 사이트를 보면 notion 페이지 주소를 입력하면 자동으로 사이트를 만들어주는 사이트입니다.
  • 사이트를 분석해본 결과, notion에서 제공하지 않는 html 태그를 추가할 수 있고 레이아웃의 배치를 바꾸거나, 외부 라이브러리를 추가 할 수 있습니다.
  • 핵심은 노션 페이지를 어떻게 배포를 할 것인가? → 특히 oopy.io 라는 도메인을 사용해서 처리하는 것을 어떻게 하는 것인가?는 아직 좀 더 파악이 필요하다.

1-2. 동작 방식 분석

  • notion page 주소를 입력하면 그 page를 홈페이지 front로 만들어 줌
  • 내가 봤을 때 notion page 입력받은 주소를 html로 변경을 해줌 → 이 html 파일을 웹 front로 띄워주는 역할 하는 것.
  • 그리고 내가 쓸 url을 입력하면 https:// <내가 쓸 url명>.oopy.io을 만들어 준다.
  • 일단 html을 내가 막 변경하기에 front 지식이 조금 없으니까 위 두가지만 실행되는 것을 하나 만들어보고자 한다.

1-3. 개발 목적

  • 우피 사이트를 벤치마킹해서 내가만든 노션 페이지를 github page로 제공되는 웹으로 호스팅하고자 한다.
  • 노션에 이 페이지 처럼 기획하고 개발하는 과정들을 간편하게 정리한 뒤 정리 내용을 그대로 github의 page에 호스팅하고자 한다.
  • 해당 프로젝트와 함께 index.html 파일을 올리고 git repo에서 간단한 설정만 해주면 바로 그 사이트가 보여지기 때문이다.
  • 개발 문서화 과정을 자동화하여 궁극적으로는 개발의 생산성을 좀 더 향상하고자 함이다.

1-4. 개발 기획

  • 시스템 구성
    • 입력 : 내 프로젝트 경로와 notion page 주소
    • 처리 : notion page 주소를 입력받아 해당 페이지를 html로 변환한다. 그리고 변환한 html을 프로젝트 경로에 저장한다.
    • 출력 : 프로젝트 경로에 index.html
  • 사용자 동작순서
    1. app.js를 실행한다.
    2. notion page 주소와 내 프로젝트 경로를 입력하고 제출 버튼을 누른다.
    3. 프로젝트 경로에서 index.html 파일을 확인한다.
    4. 프로젝트를 git에 push한다.
    5. git repo에서 page 세팅을 해준다.
  • 개발 계획
    • 기술 스택
      • nodejs, git, bootstrap
    • 개발 수준
      • git에서 해당 프로젝트를 내려받고, 해당 프로젝트에서 app.js를 실행한 뒤 위 사용자 동작순서를 거치는 방법
    • 향후 개선안
      • 위 방식대로라면 local에서만 사용이 가능한데, 차후 배포가 가능한 형태로 개선
      • index.html 생성과 git push가 별도로 동작하는데, 한 번에 동작할 수 있도록 개선
      • 기타 로직상 버그 제거 ( nodejs 미숙으로 인하여 )

2. 개발

2-1. 개발환경 구축

  • node version : v16.14.2
  • bootstrap : 4.6.1
  • 사전 설정
    • git 설정 ( github repo pages 설정 )
    • notion 설정 ( nodejs 모듈 사용 가능 설정 )

2-2. 개발

  • bootstrap을 사용하여 간단한 front page 작성
  • nodejs의 notion-page-to-html 모듈을 사용하여 notion page 변환 및 프로젝트 경로에 파일 저장하는 로직 개발
  • 테스트 코드 작성

** 각각의 내용에 대해서는 기술 블로그에 세부 내용을 자유롭게 작성하고 링크만 최종적으로 걸어두도록 함. **

3. 테스트

3-1. 내용

  • 단위 테스트 : 각 모듈별 작성한 테스트 코드를 별도 ‘테스트명.js’ 파일을 실행하여 입력과 출력에 대한 결과 확인
  • 통합테스트 : 사용자 동작 순서에 따른 통합테스트 진행

    ** 간단한 프로그램이라 자세한 문서화는 생략하고 프로그램 주석에 내용을 작성하도록 함 **
